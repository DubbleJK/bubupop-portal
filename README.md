# 부부계산 포털 (bubucalculate-app)

사무실에서 쓰는 도구를 한 곳에서 열 수 있는 포털입니다.

- **견적 계산기** → 별도 앱(quotation-app)으로 연결
- **키워드 검색량 · 연관/인기 키워드** → 키워드 입력 시 모바일/PC 검색 트렌드, 연관키워드 15개, 인기키워드 15개 (`/keyword`)
- **네이버 SEO 블로그 작성** → 키워드·업종 입력으로 블로그 글 자동 생성 (`/blog`)

## 실행 방법

```bash
# 의존성 설치
npm install

# 개발 서버 (포트 3001)
npm run dev
```

브라우저에서 **http://localhost:3001** 로 접속합니다.

## 견적 계산기 함께 쓰기

1. **quotation-app** 터미널에서: `cd quotation-app` → `npm run dev` (포트 3000)
2. **bubucalculate-app** 터미널에서: `cd bubucalculate-app` → `npm run dev` (포트 3001)
3. 브라우저에서 **http://localhost:3001** 로 포털 접속 후, "견적 계산기" 카드를 누르면 견적 앱(3000)이 새 탭에서 열립니다.

## 환경 변수

- `OPENAI_API_KEY` — 블로그 글 생성 + 키워드 연관/인기 추천에 사용. (필수: 블로그·키워드 기능 쓰려면)
- `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET` — 네이버 데이터랩(검색어 트렌드) API. 있으면 키워드 조회 시 **PC/모바일 검색 트렌드**(상대값 0~100)를 표시. 없으면 트렌드는 비고, 연관/인기 키워드는 OpenAI로만 생성.
  - 발급: [네이버 개발자 센터](https://developers.naver.com/apps) → 애플리케이션 등록 → 사용 API에서 **데이터랩(검색어트렌드)** 선택.
- `NEXT_PUBLIC_QUOTATION_APP_URL` — 견적 계산기 링크. 기본값: `http://localhost:3000`. 배포 시 견적 앱 URL로 설정.

## 블로그 글 생성기 (처음 쓰시는 분)

**코딩 경험이 없어도** 실행할 수 있도록 단계별 설명을 적어 두었습니다.  
→ **블로그글생성_실행방법.md** 파일을 열어서 1단계부터 따라 하시면 됩니다.

## 배포 시 한 웹사이트에서 모두 쓰는 방법

**나중에 배포할 때** 포털·블로그·견적을 **한 웹사이트(한 주소)**에서 쓰고 싶다면 아래 두 가지 중 하나로 하면 됩니다.

### 방법 1: 두 개 배포 후 포털에서 링크 (가장 간단)

- **포털(bubucalculate-app)** 을 Vercel 등에 배포 → 예: `https://내포털.vercel.app`
- **견적(quotation-app)** 을 따로 한 번 더 배포 → 예: `https://내견적.vercel.app`
- 포털 프로젝트의 환경 변수에 `NEXT_PUBLIC_QUOTATION_APP_URL=https://내견적.vercel.app` 설정
- 사용자는 **포털 주소 하나**만 알면 되고, 견적 계산기 카드를 누르면 견적 사이트가 새 탭에서 열림  
→ “한 곳(포털)에서 시작해서 모두 사용” 가능합니다.

### 방법 2: 견적 앱을 포털에 합쳐서 한 도메인에 (진짜 한 웹사이트)

- 견적 계산기 코드(페이지·컴포넌트)를 **이 포털 프로젝트 안으로** 옮겨서 `/quote` 경로로 서비스
- 그러면 한 번 배포한 사이트에 **`/`(포털), `/blog`, `/quote`(견적)** 가 모두 있음  
→ **한 웹사이트(한 도메인)**에서 모두 가능합니다.  
- 합치는 작업이 필요하므로, 배포 시점에 “견적을 포털에 합쳐서 한 사이트로 배포하고 싶다”고 하시면 그때 단계별로 정리해 드릴 수 있습니다.

**정리**: 배포할 때도 **한 웹사이트에서 모두 가능**합니다. 방법 1은 설정만으로, 방법 2는 견적 코드를 포털에 합치면 됩니다.

## 네이버 SEO 블로그 작성 (SEO 반영 여부)

블로그 글 생성기는 **네이버 검색 노출을 고려한 SEO**에 맞게 글을 쓰도록 프롬프트를 넣어 두었습니다.

- **제목**: 메인 키워드를 앞쪽에, 25~35자 내외 (네이버 검색 노출에 유리하도록 안내)
- **서론**: 첫 1~2문장 안에 메인 키워드 자연스럽게 포함
- **본문**: 서브 키워드와 연관 표현을 고르게 배치, 키워드 과다 반복(스터핑)은 하지 않도록 지시
- **구성**: 정보형 + 문의 유도형, FAQ로 검색 질의에 대응
- **해시태그**: 메인·서브 키워드, 지역, 업종 관련 검색어 포함

실제 노출·순위는 네이버 정책과 경쟁 환경에 따라 달라질 수 있지만, **SEO에 근거한 글쓰기 규칙**은 적용된 상태입니다.

## 프로젝트 구분

- **bubucalculate-app** (이 프로젝트): 포털 대시보드 + 네이버 SEO 블로그 글 생성.
- **quotation-app**: 인쇄 견적 계산기. 이 프로젝트와 분리되어 있어, 포털에 문제가 생겨도 견적 앱은 그대로 사용할 수 있습니다.
