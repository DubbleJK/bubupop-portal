# 변경 사항 적용하기 (배포까지)

코드 수정 후 **로컬에서 적용**하고 **Vercel에 반영**하는 방법입니다.

---

## 1. 포털 (bubucalculate-app) 적용

터미널(PowerShell 또는 Cursor 터미널)에서 아래를 **순서대로** 실행하세요.

```powershell
# 1) 포털 폴더로 이동
cd C:\Users\USER\bubucalculate-app

# 2) 변경된 파일 확인 (선택)
git status

# 3) 변경 파일 모두 스테이징
git add .

# 4) 커밋 (메시지는 원하는 대로 수정)
git commit -m "포털 업데이트"

# 5) GitHub로 푸시 → Vercel이 자동으로 포털 다시 배포
git push origin main
```

- **5번**까지 끝나면 Vercel에서 포털 프로젝트(bubupop-portal)가 자동으로 다시 빌드·배포됩니다.
- Vercel 대시보드 → **Deployments**에서 진행 상황을 확인할 수 있습니다.

---

## 2. 견적 계산기 (quotation-app) 적용

포털이 아니라 **견적 계산기** 쪽만 수정했을 때는 아래처럼 합니다.

```powershell
# 1) 견적 앱 폴더로 이동
cd C:\Users\USER\quotation-app

# 2) 변경 확인 (선택)
git status

# 3) 스테이징 → 커밋 → 푸시
git add .
git commit -m "견적 앱 업데이트"
git push origin main
```

- 푸시하면 Vercel의 **견적 계산기 프로젝트(bubupop-quotation)** 가 자동으로 다시 배포됩니다.

---

## 3. 환경 변수만 바꾼 경우 (Vercel)

**.env.local**은 로컬에만 있고 Git에 안 올라가므로, **Vercel에서 키만 추가/수정**했을 때는:

1. Vercel 대시보드 → 해당 프로젝트 → **Deployments**
2. 맨 위 배포 오른쪽 **⋮** → **Redeploy**

재배포하면 새 환경 변수가 적용됩니다. (코드 푸시 없이 Redeploy만 해도 됩니다.)

---

## 요약

| 수정한 곳        | 이동할 폴더           | 푸시 후 자동 배포되는 곳   |
|-----------------|------------------------|----------------------------|
| 포털            | `bubucalculate-app`    | bubupop-portal (Vercel)    |
| 견적 계산기     | `quotation-app`        | bubupop-quotation (Vercel) |

**적용 순서:** `cd` → `git add .` → `git commit -m "메시지"` → `git push origin main`
