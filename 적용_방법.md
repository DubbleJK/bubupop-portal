# 변경 사항 적용하기 (배포까지)

코드 수정 후 **로컬에서 적용**하고 **Vercel에 반영**하는 방법입니다.

---

## 1. 포털 (bubucalculate-app) 적용

터미널(PowerShell 또는 Cursor 터미널)에서 아래를 **순서대로** 실행하세요.

```powershell
# 1) 포털 폴더로 이동
cd C:\Users\USER\bubucalculate-app

# 2) 변경된 파일 확인 (선택)
git status

# 3) 변경 파일 모두 스테이징
git add .

# 4) 커밋 (메시지는 원하는 대로 수정)
git commit -m "포털 업데이트"

# 5) GitHub로 푸시 → Vercel이 자동으로 포털 다시 배포
git push origin main
```

- **5번**까지 끝나면 Vercel에서 포털 프로젝트(bubupop-portal)가 자동으로 다시 빌드·배포됩니다.
- Vercel 대시보드 → **Deployments**에서 진행 상황을 확인할 수 있습니다.

---

## 2. 견적 계산기 (quotation-app) 적용

포털이 아니라 **견적 계산기** 쪽만 수정했을 때는 아래처럼 합니다.

```powershell
# 1) 견적 앱 폴더로 이동
cd C:\Users\USER\quotation-app

# 2) 변경 확인 (선택)
git status

# 3) 스테이징 → 커밋 → 푸시
git add .
git commit -m "견적 앱 업데이트"
git push origin main
```

- 푸시하면 Vercel의 **견적 계산기 프로젝트(bubupop-quotation)** 가 자동으로 다시 배포됩니다.

---

## 3. 발주서 발행기 (purchase-order-app) 적용

**발주서 발행기**만 수정했을 때는 아래처럼 합니다.

```powershell
# 1) 발주서 앱 폴더로 이동
cd C:\Users\USER\purchase-order-app

# 2) 변경 확인 (선택)
git status

# 3) 스테이징 → 커밋 → 푸시
git add .
git commit -m "발주서 발행기 업데이트"
git push origin main
```

- 푸시하면 Vercel의 **발주서 발행기 프로젝트(bubupop-order)** 가 자동으로 다시 배포됩니다.
- **처음 한 번만:** GitHub 저장소 생성 + Vercel 프로젝트 연결이 필요합니다. → 아래 "발주서 발행기 처음 배포하기" 참고.

---

## 4. 환경 변수만 바꾼 경우 (Vercel)

**.env.local**은 로컬에만 있고 Git에 안 올라가므로, **Vercel에서 키만 추가/수정**했을 때는:

1. Vercel 대시보드 → 해당 프로젝트 → **Deployments**
2. 맨 위 배포 오른쪽 **⋮** → **Redeploy**

재배포하면 새 환경 변수가 적용됩니다. (코드 푸시 없이 Redeploy만 해도 됩니다.)

## 발주서 발행기 처음 배포하기 (한 번만)

발주서 발행기(purchase-order-app)는 아직 Git 저장소가 없을 수 있습니다. **처음 한 번** 아래 순서로 진행하세요.

1. **GitHub에서 새 저장소 생성**
   - GitHub → New repository → 이름 예: `bubupop-order` (또는 `bubupop-purchase-order`)
   - Public, README 없이 생성해도 됨.

2. **로컬에서 Git 초기화 및 푸시**
   ```powershell
   cd C:\Users\USER\purchase-order-app
   git init
   git add .
   git commit -m "발주서 발행기 초기 커밋"
   git branch -M main
   git remote add origin https://github.com/DubbleJK/bubupop-order.git
   git push -u origin main
   ```
   (저장소 이름이 다르면 `git remote add origin` URL만 본인 저장소 주소로 바꾸세요.)

3. **Vercel에서 프로젝트 추가**
   - Vercel 대시보드 → Add New Project → Import Git Repository
   - 방금 만든 `bubupop-order` 저장소 선택
   - Framework Preset: **Vite** (또는 자동 감지)
   - Build Command: `npm run build`, Output Directory: `dist` (vercel.json에 이미 있으면 자동 적용)
   - Deploy 후 배포 URL 확인 (예: https://bubupop-order.vercel.app)

4. **포털에서 링크 연결**
   - 포털(bubucalculate-app) 메인에 "발주서 발행기" 카드가 있습니다.
   - 배포 URL을 사용하려면 포털의 Vercel 환경 변수에 `NEXT_PUBLIC_ORDER_APP_URL` = `https://bubupop-order.vercel.app` 추가 후 Redeploy 하면 됩니다.
